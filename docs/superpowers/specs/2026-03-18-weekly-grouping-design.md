# 已完成任务按周分组功能设计

## 需求背景
用户希望能够按周查看已完成的任务，以便了解每周的工作完成情况。

## 设计目标
- 将已完成的任务按自然周分组展示
- 本周任务默认展开，历史周默认折叠
- 界面清晰，易于浏览和定位特定周的任务

## 设计方案

### 周定义
- 自然周：周一到周日（ISO 8601 标准）
- 分组依据：任务的 `finishedAt` 时间戳

### 标签显示格式
| 周类型 | 显示格式 | 示例 |
|--------|----------|------|
| 当前周 | 本周 (M月D日 - M月D日) | 本周 (3月16日 - 3月22日) |
| 前一周 | 上周 (M月D日 - M月D日) | 上周 (3月9日 - 3月15日) |
| 前两周 | 两周前 (M月D日 - M月D日) | 两周前 (3月2日 - 3月8日) |
| 更早 | YYYY年M月D日 - M月D日 | 2025年3月16日 - 3月22日 |

### UI 结构
```
已完成任务区域
├── 本周分组 (默认展开)
│   ├── 任务1
│   ├── 任务2
│   └── ...
├── 上周分组 (默认折叠)
├── 两周前分组 (默认折叠)
└── 更早周分组 (默认折叠)
```

### 交互行为
1. 点击分组标题可展开/折叠该周的任务列表
2. 展开/折叠状态不持久化，每次打开应用恢复默认设置
3. 分组按时间倒序排列（本周在最上）
4. 每个分组显示该周完成的任务数量

### 数据层变更

#### TaskDriver 新增方法
```dart
/// 将已完成任务按周分组
/// 返回 Map: 周标签 -> 该周的任务列表
Map<String, List<Task>> groupTasksByWeek(List<Task> finishedTasks)
```

#### 周计算辅助方法
```dart
/// 获取任务所在的周标识（用于分组）
String getWeekKey(DateTime finishedAt)

/// 生成周标签
String getWeekLabel(DateTime weekStart, {bool isRelative = true})
```

### UI 层变更

#### 新增组件
- `WeekGroupHeader`: 周分组标题组件，显示标签和展开/折叠状态

#### 修改组件
- `TodoList`:
  - 使用分组后的数据渲染已完成任务
  - 维护各周的展开/折叠状态（`Map<String, bool>`）
  - 本周默认展开，其余默认折叠

### 边界情况处理
1. **没有已完成任务**: 显示"暂无已完成任务"提示
2. **某周没有任务**: 不显示该周分组
3. **跨周完成任务**: 严格按照 `finishedAt` 时间戳分组
4. **时区问题**: 使用设备本地时区计算周
